Review Findings

ID Severity Finding Evidence Required Plan
SEC-01 Critical Local-admin MySQL bootstrap requires fresh-host validation. setup.sh now installs/uses MySQL Server binaries, writes socket-only etc/my.cnf, initializes .data/mysql, starts mysqld, and renames root@localhost to the local admin account. Validate this flow on OL8, OL9, Ubuntu, macOS, and OCI Compute images before treating it as release-complete.
SEC-02 High Uploaded SSH key profiles are not fully wired to runtime connections. modules/profiles.py stores uploaded keys under profile_ssh_keys/, but modules/mysql_connection.py still requires profile["ssh_key_path"] for SSH tunnels. Resolve uploaded keys server-side from ssh_key_id, never render the path, and keep legacy explicit paths disabled unless an operator enables them.
SEC-03 High Dependency audit found a Paramiko vulnerability. .venv/bin/python -m pip_audit -r requirements.txt initially reported Paramiko affected by CVE-2026-44405. PyPI 4.0.0 was still reported vulnerable. Pin Paramiko to the upstream patched commit until a fixed PyPI release is available, and keep a strict audit gate for release builds.
SEC-04 High Platform MySQL Shell installers are too shallow for fresh hosts. ol8 and ol9 scripts call dnf install mysql-shell directly. ubuntu runs apt-get install mysql-shell without the current MySQL APT signing key/source setup. Align per-OS installers with the myapp reference: OL8 module disablement, Oracle MySQL repo setup, Ubuntu stale-source cleanup, key installation, and innovation repo configuration.
SEC-05 Medium Profile setup script accepts arbitrary output paths. secured_connection_profile_setup.sh validates profile names but does not reject path traversal or unexpected absolute output paths when called directly. Add path validation or explicit --allow-absolute-path semantics. Keep normal setup invocation pinned to repo-local paths.
SEC-06 Medium Flask secret key is not yet generated by setup. app.py still has a static fallback: mysql-shell-web-change-me. Generate .flask_secret_key in setup, persist FLASK_SECRET_KEY through runtime env, and chmod the file 0600.
SEC-07 Medium Systemd capability grant is broader than necessary. Generated services always set AmbientCapabilities=CAP_NET_BIND_SERVICE, even when configured ports are above 1024. Render the capability only when the selected HTTP or HTTPS port is privileged.
SEC-08 Medium Update polling still has a signed-cookie fallback. Status authorization supports the custom header but also accepts a token from Flask session. Keep authenticated app-shell access, but require the custom header for restart-survival token reads.
SEC-09 Low Version history exists only as appver.json and README prose. appver.json contains 1.0.4 and version_history.json now records release history. Keep version history updates as a release gate whenever appver.json changes.

Controls Already Aligned

  • Login page renders only profile names and default username metadata, not hostnames, sockets, SSH hosts, or key paths.
  • Profile management routes are gated by authenticated local-admin-profile.
  • profiles.json is removed from source tracking and replaced with profiles.example.json.
  • .gitignore excludes runtime state, uploaded keys, TLS keys, generated MySQL config, security reports, and token-like files.
  • Repository version checks use verified TLS and normalize GitHub raw URLs to the contents API.
  • Auto-update passes local-admin bootstrap password only through worker environment variables, not command-line arguments.

OCI Compute Init-Script Work Plan

OCI Compute deployment must use platform-specific init-script wrappers and validation. Oracle Linux images use the opc login user by default, while Ubuntu images use ubuntu. The init script must pass local-admin bootstrap values through environment variables and must never run with shell tracing that would print passwords into /var/log/mysql-shell-web-init.log.

Remediation Plan

Phase 1: Complete Local-Admin MySQL Bootstrap

  • Add platform package installation for MySQL Server binaries in setup.sh.
  • Write app-local etc/my.cnf with skip-networking, mysqlx=0, and repo-local paths.
  • Initialize .data/mysql with mysqld --initialize.
  • Read the temporary root password from .data/log/mysqld.err.
  • Start the app-managed MySQL process only over the Unix socket.
  • Rename root@localhost to the supplied local admin user and set the temporary password.
  • Run secured_connection_profile_setup.sh after the database is initialized.

Phase 2: Harden Profile and SSH Key Runtime

  • Resolve uploaded SSH keys from ssh_key_id on the server side.
  • Remove browser-rendered and source-controlled key paths from the default workflow.
  • Add direct tests for path traversal rejection and atomic key replacement.
  • Tighten secured_connection_profile_setup.sh path validation.

Phase 3: Align Platform Installers

  • Update OL8 and OL9 installer scripts for Oracle MySQL repository setup and OL8 module disablement.
  • Update Ubuntu installer for current MySQL APT key/source recreation and stale source cleanup.
  • Add macOS embedded MySQL Server setup for local-only socket bootstrap.
  • Keep embedded Linux/macOS paths out of OCI Compute docs unless platform-specific.

Phase 4: Security and Update Hardening

  • Generate .flask_secret_key during setup and export FLASK_SECRET_KEY.
  • Render AmbientCapabilities=CAP_NET_BIND_SERVICE only when a selected service port is below 1024.
  • Require the update poll token in X-MySQL-Shell-Web-Update-Poll-Token for unauthenticated restart-window reads.
  • Pin or mitigate Paramiko after validating the advisory fix path.
  • Run pip-audit in warn-only setup mode and strict release mode.

Phase 5: Version History and Release Gate

  • Add VERSION_HISTORY.md or version_history.json.
  • Record security, deployment, auto-update, and compatibility changes for every release.
  • Require version history update whenever appver.json changes.
  • Add release checks for syntax, login secrecy, route authorization, dependency audit, and platform dry-run validation.

Phase 6: OCI Compute Init-Script and Old-Updater Compatibility

  • Maintain one generic oci_compute_init.sh with platform defaults and no shell tracing.
  • Add README user-data examples for OL8, OL9, and Ubuntu as separate code blocks.
  • Require explicit LOCAL_MYSQL_ADMIN_PASSWORD for first-boot local-admin bootstrap.
  • Pass local-admin password only through process environment, never command arguments or logs.
  • Keep old Auto-Update forms compatible by treating missing bootstrap fields as code-refresh-only runs.
  • After restart, require the refreshed Auto-Update page to collect temporary local-admin credentials and complete bootstrap.

Decision Log

  • Keep profiles.json local-only. Use profiles.example.json for safe starter metadata.
  • Use socket-only local-admin profile as the sole profile-management authority.
  • Do not store database passwords, private-key contents, private-key passphrases, API tokens, or DSNs in profile files.
  • Use verified TLS for repository version checks. Do not disable certificate verification.
  • Use full setup with sudo when available; keep SKIP_PRIVILEGED_SETUP=1 as a restricted updater fallback.

MySQL Shell

Load Validation

Load validation focuses on target identity, GTID state, and load option profile safety before loadDump runs.

Target Checks

The loadDump page displays the target DB System information before execution.

  • DB System hostname and version
  • server UUID and server ID
  • GTID mode
  • gtid_executed and gtid_purged
  • other server UUIDs found in GTID state

GTID Rules

Empty GTID Green

No executed or purged GTID is present.

Local GTID Info

GTID only references the target server UUID.

Mixed GTID Red

GTID includes a different server UUID.

A mixed GTID warning means the target may already contain transactions from another server. Review the target before loading data.

Option Profile Checks

The load option profile editor warns and asks for confirmation when risky settings are saved.

  • loadUsers is not checked.
  • updateGTIDSet is off.
  • resetProgress is off.

Mitigation

Mixed GTID state

Confirm the target DB System is the intended restore target. If the target should be empty, inspect existing data and GTID state before running loadDump.

updateGTIDSet

Choose the MySQL Shell updateGtidSet mode that matches the restore plan. Keep it off only when you intentionally do not want the dump GTID state applied.

resetProgress

Enable resetProgress for a fresh load. Leave it off only when intentionally resuming from an existing progress file.

loadUsers

Enable loadUsers when user accounts from the dump should be restored. Leave it off only when target accounts are managed separately.

Dump Validation

Dump validation reviews the selected dump option profile before dumpInstance or dumpSchemas runs.

Validation Scope

Validation is enabled after a dump option profile is selected. The checks use the effective profile filters, not every table in the DB System.

  • dumpInstance starts from all accessible user schemas.
  • dumpSchemas starts from the schemas selected on the operation page.
  • includeSchemas narrows the schema scope.
  • excludeSchemas removes schemas from the scope.
  • includeTables narrows validation to those fully qualified tables.
  • excludeTables removes those fully qualified tables from validation.
  • System schemas and mysql_% schemas are hidden and ignored.

Validation Rules

Primary Key Required

Tables without a primary key are flagged.

Engine InnoDB

Non-InnoDB engines are flagged.

Lakehouse Excluded

Lakehouse tables are not supported.

Events Review

Enabled events are flagged.

Auth Plugin Review

mysql_native_password is flagged.

Charset utf8mb4

Table and column charset drift is flagged.

Mitigation

Primary key check

Open Admin > DB Admin, review the primary-key section, and apply the suggested fix where appropriate. The app can add an invisible auto-increment primary key for eligible tables, or reuse an existing auto-increment column when one exists.

Lakehouse tables

Enable Exclude Lakehouse tables in the dump option profile. The app adds in-scope Lakehouse tables to excludeTables.

Events

Disable events before dumping, exclude event objects in the dump profile, or confirm that restoring enabled events into the target DB will not change data unexpectedly.

Authentication plugins

Review accounts using mysql_native_password and migrate them to a supported authentication plugin before relying on future MySQL versions.

Character set and collation

Review tables and columns that differ from utf8mb4 and utf8mb4_0900_ai_ci. Convert only after application compatibility has been checked.

Lakehouse Filtering

The Lakehouse count and the generated exclusion list follow the active dump profile filters.

  • Lakehouse detection includes tables with ENGINE=Lakehouse and tables marked with secondary_engine=rapid.
  • If includeTables is set, only Lakehouse tables in that include list are counted.
  • If includeSchemas is set, only Lakehouse tables in those schemas are counted.
  • If excludeTables already removes a Lakehouse table, it is not counted again.
  • If Exclude Lakehouse tables and includeTables overlap, validation shows an option conflict and the run is blocked until the overlap is resolved.

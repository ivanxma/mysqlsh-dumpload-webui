1. Login and Connection Profile Workflow

Control Current Behavior Status Evidence
Login profile secrecy Login form uses saved profile names only. It does not render host, port, socket, SSH jump host, or key path values before authentication. Pass Flask test client leak scan returned an empty leak list.
Profile selection User must select a saved profile. The app loads the full profile server-side after form submission. Pass public_login_profiles() exposes profile name and default username only.
Profile management authorization /admin/profile is protected by local-admin authentication. Non-authenticated requests redirect. Pass Unauthenticated GET returned HTTP 302 to /.
Uploaded SSH key path secrecy Uploaded keys are stored under profile_ssh_keys/<profile-id>/id_key and resolved server-side by key id. Pass resolve_stored_ssh_key_path() is used by MySQL SSH tunnel setup.

Fix Plan

  • Add regression checks that login HTML never contains host, socket, ssh_host, or key path values from profiles.json.

2. Secure Connection Profile and Embedded MySQL Initialization

Control Current Behavior Status Evidence
Socket-only local admin profile local-admin-profile is created as mode=socket and marked for first-login password rotation. Pass secured_connection_profile_setup.sh writes non-secret socket profile metadata and chmods profile storage to 0600.
App-managed MySQL config setup.sh writes generated etc/my.cnf with repo-local datadir, socket, PID, log, tmp path, skip-networking, and mysqlx=0. Code Verified Shell syntax passed. Fresh-host MySQL runtime validation still required.
Embedded/app-managed initialization setup.sh initializes .data/mysql, extracts the temporary root password, starts mysqld on the socket, renames root@localhost to the local admin user, and grants local admin privileges. Host Test Needed Implementation exists in initialize_local_mysql_if_needed(). Not exercised on a fresh OL/Ubuntu/macOS host in this local report.
Localadmin password reset First login through local-admin-profile redirects to the password-change page when force_password_change=true. After change, session state is cleared and the user must log in again. Code Verified Change-password flow is limited to socket-mode local admin profile.
Flask session secret Setup generates .flask_secret_key, exports FLASK_SECRET_KEY_FILE, and start scripts pass it into the app. Pass Shell syntax passed and runtime env/start scripts include the secret file variable.

Fix Plan

  • Run fresh-host bootstrap on OL8, OL9, Ubuntu, and macOS and capture socket-only MySQL proof.
  • Verify no TCP listener for local admin MySQL with ss -ltnp or platform equivalent.
  • Verify first login with temporary localadmin password forces password reset and logs out after reset.
  • Add automated smoke test that checks profiles.json contains no password, token, DSN, private key content, or private-key passphrase.
